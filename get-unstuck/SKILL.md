# get-unstuck

Get a stalled task moving by finding *why* it's stuck, not by cheerleading. Low motivation is usually a signal: the task is vague, too big, blocked on an unknown, or carries a dread that has a specific cause. Locate the friction, then shrink the task to a first step small enough that starting is easier than avoiding. No hype, no "you've got this" - just the mechanics of momentum.

## When to use this

When someone knows what to do but isn't doing it. Procrastination, dread, or overwhelm on a task that's genuinely theirs to do. Not for tasks that are actually the wrong thing to do - if avoidance is a valid signal the task shouldn't happen, say that instead.

## Input

If the task they're stalled on was given with the invocation, take it and work the steps below. Only ask the user what they're stuck on when the invocation carried nothing.

## Steps

1. **Find the actual friction.** Ask what specifically makes this heavy. Usually one of: it's too vague to start, it's too big to hold in your head, it's blocked on something unknown, or it carries dread with a nameable cause (fear of finding out it's broken, an unpleasant conversation, etc.). Name which one - the fix differs for each.

2. **Address that specific friction:**
   - *Too vague* → sharpen it to a concrete outcome.
   - *Too big* → find the natural first slice, ignore the rest for now.
   - *Blocked on unknown* → the real first task is answering the unknown, not the main work.
   - *Dread* → name what's actually feared; it's usually smaller and more specific than the ambient weight suggests.

3. **Shrink to a two-minute start.** Name one action small and concrete enough that doing it is less effort than continuing to avoid it. Not "write the report" - "open the doc and write the one-line summary at the top." Starting breaks the stall; the rest usually follows once moving.

4. **Hand over the single step.** End with exactly one action to take right now. No plan, no phases, no motivational padding - just the next physical move.

## Rules

- Diagnose the friction before prescribing. The step depends on *why* it's stuck.
- No hype, encouragement theater, or "you've got this". Momentum comes from a small enough step, not a pep talk.
- End with exactly one concrete action, sized to be done in a couple of minutes.
- If avoidance is a valid signal the task is wrong or misprioritized, say that - don't push someone into the wrong work.

## Example

User: "I've been putting off writing the migration guide for two days."

Weak response (hype): "You can do it! Just block out some time and power through!"

Strong unstick:
> **Friction:** Which is it - you don't know what to cover (vague), it feels enormous (too big), or you're unsure the migration steps even work (blocked/dread)?
> Assuming it's *too big*: you're picturing the whole guide. Don't. The natural first slice is a bare outline - just the section headings for each migration step, no prose.
> **Start here:** Open the file and write the heading list, nothing else. Five minutes. Once the shape exists, filling sections is a different, lighter task than staring at a blank page.
